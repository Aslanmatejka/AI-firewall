Quick Start (read this first)

You are looking at the dashboard. If the badge top-right says Firewall Active, you are done installing.

If the service is NOT running yet

  1. Open PowerShell in the project folder
  2. Run: .\scripts\install.ps1 (first time only)
  3. Run: .\scripts\start.ps1
  4. Keep that window open. Refresh this page.

The only three policies you need to know

PolicyWhat it does
allowAI gets access silently.
askYou must click Allow or Deny on the Dashboard (Pending Approvals).
blockDenied automatically.

When unsure, use ask. That is the default.

Desktop app (optional)

.\scripts\launch-dashboard.ps1

Requires the service above to be running first.

Full written guides live in the project docs/ folder: INSTALLATION.md, USER_GUIDE.md, TROUBLESHOOTING.md.

Pending Approvals

When policy is ask, a yellow panel appears on the Dashboard.

  1. Read which app wants access (e.g. Cursor → network → api.openai.com).
  2. Click Allow or Deny.
  3. If you wait 2 minutes, it auto-denies (blocks).

No popup outside the browser? That is normal — check the Dashboard page, not Windows toast notifications.

Policies

Every resource AI can access is governed by one of three policies:

PolicyBehavior
allowAccess granted automatically, logged to audit trail.
askPrompts you with Allow / Deny before access proceeds.
blockAccess denied immediately, no prompt shown.

Policies apply per-folder, per-app, and globally (network, clipboard, microphone, camera).

User Actions

AI Processes

  • Block — set app policy to block all resources
  • Allow — trust the app across files, network, clipboard
  • Stop — terminate the running process

Network

  • Block Domain — add Windows Firewall rule for an AI provider
  • Block All AI Domains — firewall rules for all 16+ known providers

Protected Files

  • Change per-folder policy via dropdown
  • Add custom folders with the + Add Folder button (desktop)

Troubleshooting

ProblemFix
Service OfflineRun .\scripts\start.ps1 and keep PowerShell open.
No pending requestsPolicy may be allow, or no AI app is running.
Block Domain failsRestart PowerShell as Administrator.
Clipboard warning in logsHarmless — poll mode still works.

Run python scripts\test-live-approval.py to verify approvals work.

Components

  • Process Monitor — polls every 2s for known AI signatures
  • AI Detector — finds local .gguf/.safetensors models and GPU workloads
  • Network Firewall — tracks TCP connections to AI endpoints
  • File Access Guard — directory watchers on protected paths
  • Clipboard Guard — sensitive pattern detection (passwords, API keys)
  • Browser Protection — scans Chrome, Edge, Firefox for AI extensions
  • Permission Manager — SQLite audit log and per-app grants

API Reference

The service exposes a REST API at http://127.0.0.1:9470:

EndpointDescription
GET /api/statusLive firewall status, processes, connections
POST /api/process/blockBlock an AI app by name
POST /api/process/terminateStop a process by PID
POST /api/policy/folderSet folder policy (allow/ask/block)
POST /api/policy/globalSet network or clipboard policy
POST /api/firewall/block-allBlock all AI provider domains
POST /api/approve/{id}Allow or deny a pending request

Administrator Mode

Some actions require elevated privileges:

  • Windows Firewall domain blocking (netsh advfirewall)
  • Hosts file modification for AI website blocking
  • Terminating protected system processes

Run AI Firewall as Administrator for full network blocking capability.
